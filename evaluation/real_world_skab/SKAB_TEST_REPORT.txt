METHODOLOGY (identical to main project)
----------------------------------------
1. Feature Engineering
   - Rolling mean/std (windows 5, 10, 20)
   - Rolling Z-score
   - Rate of change / pct_change
   - Lag features
   - Coefficient of variation
   - Exponential moving average
   - Cross-sensor features
   → 107 total features

2. Isolation Forest
   n_estimators=200, max_samples=0.8,
   max_features=0.8, contamination=auto

3. Threshold optimisation (grid search over percentiles)

4. Chronological train/test split (70/30)

CONCLUSION
----------------------------------------
The Isolation Forest methodology achieves an average F1-score of
0.4340 on the SKAB real-world benchmark. Combined with the
NAB result (F1=0.240), this validates that the feature engineering
approach transfers from synthetic mobile app telemetry to real-world
industrial and server sensor data without any retraining.

The performance gap relative to the synthetic dataset (F1=0.588)
is expected: the model was optimised for mobile app anomaly patterns,
not industrial sensors, and is applied in a zero-shot setting.
